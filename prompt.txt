I want to build a standalone CLI tool called `lba-midi-play` that plays 
MIDI music directly from LBA1's original `midi_mi.hqr` file using 
TinySoundFont, TML, and miniaudio — all single-header C libraries.

The local twin-e engine repo is at: ~/twin-e  (or adjust path)
Please read the HQR reading code from that repo first so we use the 
real implementation, not a guess.

See Resources below for file locations.

## Steps

1. Read ~/twin-e/src/hqr.c and ~/twin-e/src/hqr.h to understand the 
   existing HQR format parsing, then use that logic directly.

2. Create a new directory `lba-midi-play/` with this structure:
   - main.c
   - hqr.c + hqr.h  (adapted from twin-e, minimal dependencies)
   - xmidi.c + xmidi.h  (convert XMIDI from midi_mi.hqr to standard 
     SMF in memory — reference ScummVM's midiparser_xmidi.cpp logic 
     but implement in C)
   - Makefile

3. Download these headers into lba-midi-play/:
   - tsf.h from https://raw.githubusercontent.com/schellingb/TinySoundFont/main/tsf.h
   - tml.h from https://raw.githubusercontent.com/schellingb/TinySoundFont/main/tml.h
   - miniaudio.h from https://raw.githubusercontent.com/mackron/miniaudio/master/miniaudio.h
   - GeneralUser GS soundfont from 
     https://schristiancollins.com/generaluser.php
     (if download is not possible, remind me to download GeneralUser GS.sf2 
     manually and place it in lba-midi-play/)

4. Convert the soundfont to a C header:
   xxd -i "GeneralUser GS.sf2" | sed 's/unsigned char/static const unsigned char/' > soundfont.h
   (handle the filename spaces carefully)

5. Implement xmidi.c — the key function is:
   uint8_t *xmidi_to_smf(const uint8_t *xmi_data, int xmi_size,
                          int track_idx, int *out_smf_size);
   XMIDI is an IFF FORM XMID container used by Miles Sound System.
   The delta time encoding differs from standard MIDI — intervals are 
   in 1/60th second ticks not variable-length. Convert to standard 
   SMF format 0.

6. Implement main.c with:
   - Usage: ./lba-midi-play <path/to/midi_mi.hqr> <track_index>
   - With no track index, list the number of entries in the HQR
   - Load XMI entry from HQR → convert to SMF → parse with TML → 
     render with TSF → play via miniaudio
   - Press Enter to stop playback
   - Clean error messages at each stage so we can diagnose failures

7. Build and test with:
   make
   ./lba-midi-play /path/to/lba1/midi_mi.hqr 3

## Constraints
- Pure C (C99), no C++
- No external library dependencies beyond what miniaudio needs from 
  the OS (pthreads, dl, m on Linux)
- Keep each file focused — hqr.c only does HQR, xmidi.c only does 
  conversion, main.c wires everything together
- Defensive error handling at every step with clear messages
- The soundfont is embedded in the binary via soundfont.h
